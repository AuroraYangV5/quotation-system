# 利润率设置功能说明

## 功能描述
支持三种利润率设置方式：全局统一设置、单个商品设置、批量选中设置，满足不同场景需求。

## 功能说明

### 1. 全局设置
- **位置**: 预览页面左上角"全局设置"卡片
- **操作**: 拖动滑块，从 0% 到 100%
- **效果**: 所有商品无论之前设置了什么，都会统一更新为这个利润率
- **使用场景**: 第一次进来想要整体调整，或者整体改价时使用

### 2. 单个商品设置
- **位置**: 表格每行都有利润率输入框
- **操作**: 直接输入数字或者点击增减
- **效果**: 只修改当前商品的利润率，重新计算价格
- **使用场景**: 针对特殊商品单独设置不同利润点

### 3. 批量设置
- **位置**: 预览页面右上角"批量设置"卡片
- **操作**: 
  1. 在下方表格勾选需要修改的商品（复选框）
  2. 在批量设置框输入目标利润率
  3. 按下回车应用到所有选中商品
- **效果**: 只修改选中商品，未选中保持不变
- **使用场景**: 一类商品想要使用相同利润率，不需要全局改的时候

## 实现细节

### 数据结构
每个商品项都独立保存自己的利润率设置：
```typescript
interface PriceItem {
  // ...
  profitPercent: number;    // 当前商品设置的利润率
  calculatedPrice: number;  // 计算后的新价格 = originalPrice * (1 + profitPercent/100)
  selected: boolean;        // 是否选中用于批量修改
}
```

### 计算逻辑
- 每次修改利润率后立即重新计算价格
- 前端实时显示计算结果，不需要请求后端
- 生成时把所有商品的自定义设置发送给后端生成Excel

### UI交互
- 使用复选框勾选批量选择
- 当前Sheet支持一键全选/取消全选
- 显示已选中数量提示用户

## 注意事项
- 修改是即时的，不需要保存，切换Sheet不会丢失修改
- 刷新页面会丢失修改，因为数据存在sessionStorage中
- 所有修改都在浏览器本地完成，不会提前发送到后端
