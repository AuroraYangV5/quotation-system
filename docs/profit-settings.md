# 利润率设置功能说明

## 功能描述
支持三种利润率设置方式：全局统一设置、单个商品设置、批量选中设置，满足不同场景需求。

## 功能说明

### 1. 全局设置
- **位置**: 预览页面左上角"全局设置"卡片
- **操作**: 拖动滑块，从 0% 到 100%
- **效果**: 所有商品无论之前设置了什么，都会统一更新为这个利润率
- **使用场景**: 第一次进来想要整体调整，或者整体改价时使用

### 2. 单个商品设置
- **位置**: 表格每行都有利润率输入框
- **操作**: 直接输入数字或者点击增减
- **效果**: 只修改当前商品的利润率，重新计算价格
- **使用场景**: 针对特殊商品单独设置不同利润点

### 3. 批量设置
- **位置**: 预览页面右侧"批量设置利润率"卡片
- **设计**: 批量设置选择与导出选择完全分离，互不干扰
- **操作**: 
  1. 在下方表格**批量**列勾选需要修改的商品
  2. 在批量设置框输入目标利润率
  3. 点击**应用利润率到选中项**按钮
  4. 应用完成后自动清空所有批量勾选，方便下次选择
- **效果**: 只修改选中商品，未选中保持不变
- **使用场景**: 一类商品想要使用相同利润率，不需要全局改的时候

## 实现细节

### 数据结构
每个商品项都独立保存自己的利润率设置和两个独立选择状态：
```typescript
interface PriceItem {
  // ...
  profitPercent: number;        // 当前商品设置的利润率
  calculatedPrice: number;      // 计算后的新价格 = originalPrice * (1 + profitPercent/100)
  selectedForBatch: boolean;   // 是否选中用于批量修改利润率
  selectedForExport: boolean;  // 是否选中用于导出（独立选择）
}
```

### 设计理念：两个独立选择
| 选择列 | 用途 | 独立性 |
|--------|------|--------|
| 批量列 | 选中商品进行批量利润率修改 | 完全独立于导出选择 |
| 导出列 | 选中商品进行最终导出 | 完全独立于批量选择 |

这样设计更灵活：
- 可以选A组修改利润率，选B组导出
- 批量应用后自动清空批量勾选，不影响导出选择
- 用户可以自由组合，更符合实际使用场景

### 计算逻辑
- 每次修改利润率后立即重新计算价格
- 前端实时显示计算结果，不需要请求后端
- 生成时把所有商品的自定义设置发送给后端生成Excel

### UI交互
- 表格提供两列独立复选框
- 当前Sheet支持分别对批量和导出进行一键全选/取消全选
- 显示已选中数量提示用户

## 注意事项
- 修改是即时的，不需要保存，切换Sheet不会丢失修改
- 刷新页面会丢失修改，因为数据存在sessionStorage中
- 所有修改都在浏览器本地完成，不会提前发送到后端
