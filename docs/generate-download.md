# 生成报价和下载功能说明

## 功能描述
用户预览和编辑完成后，点击生成按钮，后端根据自定义的每个商品利润率生成Excel报价表，然后浏览器下载保存到本地。

## 工作流程

1. **前端收集数据**
   - **智能选择导出**: 如果有勾选选中项，只导出选中的商品；如果没有勾选，导出全部商品
   - 收集商品数据（包括每个商品自定义的利润率）
   - 收集Sheet信息（标题、日期）保留原有格式
   - 发送JSON数据到后端 `/api/generate` 接口

2. **后端生成Excel**
   - 按Sheet分组整理数据
   - 计算每个商品的最终价格（原价 × (1 + 利润率/100)）
   - 创建新Excel工作簿，设置美观样式
   - 保留原Sheet结构和标题

3. **返回文件给浏览器下载**
   - 返回Blob二进制数据
   - 前端创建URL自动触发浏览器下载
   - 用户保存到本地即可使用

## 输出格式

- 文件格式: `.xlsx` (Excel 2007+)
- 文件名称: `报价表.xlsx`
- 样式：
  - 标题加粗放大，灰色背景
  - 表头灰色背景，居中对齐
  - 奇偶行不同背景色，方便阅读
  - 自动调整列宽适应内容
  - 完整边框

## 生成内容结构

```
Sheet: [原Sheet名称]
┌───────────────────────────────────┐
│  标题 报价表                       │ 合并单元格标题行
├───────────────────────────────────┤
│  利润率: +XX%             [日期]  │ 利润率信息行
├─────────┬─────────────────────────┤
│  规格   │  类型1  类型2  类型3...  │ 表头行
├─────────┼─────────────────────────┤
│  规格1  │   价格   价格   价格...  │ 数据行
│  规格2  │   价格   价格   价格...  │
└─────────┴─────────────────────────┘
```

## 实现细节

### 后端使用openpyxl生成
- 正确处理多Sheet多表格
- 保留原有标题和日期信息
- 应用统一美观样式模板
- 自动计算列宽

### 前端下载
- 使用 `Blob` 接收二进制数据
- 使用 `window.URL.createObjectURL` 创建下载链接
- 自动触发点击下载，不需要用户额外操作

## 使用说明
1. 确认所有商品利润率设置正确
2. **选择性导出**: 如果只需要导出部分商品，在预览表格中勾选需要导出的商品
3. 点击右上角"生成并下载"按钮
4. 等待处理完成（处理中按钮显示loading）
5. 浏览器自动下载 `报价表.xlsx`
6. 打开文件检查即可使用

## 选择性导出功能

### 使用场景
- 只需要报价部分商品，不需要全部导出
- 分批次给不同客户报价，从同一个原价表中选择不同商品
- 测试生成效果，先导出少量商品验证

### 使用方法
1. **设计特点**: 导出选择与批量利润率修改选择完全分离，互不干扰
2. 在预览表格**导出**列勾选需要导出的商品
3. 顶部信息栏会显示已选中导出商品数量
4. 当前Sheet提供"全选导出"/"取消全选导出"一键操作
5. 点击生成按钮，系统只会导出你勾选的商品
6. 如果取消所有勾选，恢复为导出全部商品

### 独立性设计
批量设置选择和导出选择是两个独立的勾选列：
- 修改利润率时勾选的批量选择，不会影响哪些商品要导出
- 选择导出哪些商品，也不会影响之前的利润率修改
- 更加灵活，支持各种组合操作

## 注意事项
- 如果利润率有多个不同值，标题行会显示"自定义利润率 (多个不同值)"
- 所有自定义设置都会保留在生成的Excel中
- 生成是一次性的，可以多次生成不同版本下载
- 勾选只影响导出内容，不影响预览显示
